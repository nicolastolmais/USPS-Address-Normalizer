Questions For the Client - Group 5

Created: Feb 5, 2016. Last Revised: Feb 10, 2016. Document created by members of Team 5 and transcribed into HTML by Alexis Carroll.

Purpose

The purpose of this document is to record questions that will be presented to the client in order to gain more knowledge about the requirements for the address normalization system.

Identify Actors

1. Who will be using the system?

2. Will it be used by the public, one individual, or many users?

3. Will the user have a technical background?

4. We will have a database working with our system. Are there any other systems we will need to work with?

5. What are the specifications on the external database?

6. What entity or organization are we working for?

Scenarios

7. What should the system do with caught address errors or addresses that are not normalizable?

Use Cases

8. How will this system be run?

9. What should the system do with mixed up address fields?

10. What should the system do if there are spanish inputs or abbreviations?

Functional Requirements

11. Will there be a user interface or just a back-end system?

12. What exactly are the inputs and outputs?

13. Should the output addresses be sorted or categorized? If so, how?

14. What defines a valid input?

15. What invalid inputs are the system not responsible for handling?

16. What are the most critical functions within the application?

17. Will we be dealing with postal addressing standards, business addressing standards, or both standards?

18. Will the inputs be supplied in plain text or in a structured file such as XML or CSV?

Non-Functional Requirements

19. How many addresses will the system need to normalize at a time?

20. Will these addresses be sensitive? How secure does the system need to be?

21. What platforms should this application be able to run on?

22. Should the system be mobile friendly?

23. Are there any space requirements that the system must meet?

24. Will the system need to meet a speed requirement (processing requests in a certain amount of time)?

25. What is the timetable for this project?

26. What maintenance will need to be done on this system after deployment?

27. What level of documentation is necessary?
